Helpers that facilitate the composing of posts for both Org Mode and Markdown.

Click [here]({{ site.baseurl }}/docs/helpers-md/) to view the contents supported
or customized for Markdown.

* Gallery

You can create a gallery as usual in [[https://mmistakes.github.io/minimal-mistakes/docs/helpers/#gallery][Minimal Mistakes]]. With this plugin you have
these extra features:

1. Add a ~columns~ parameter to each gallery to control the columns of a gallery
   such as:

   {% raw %}
   #+begin_src text
   {% include gallery columns=6 %}
   #+end_src
   {% endraw %}

   The image size should be adjusted automatically.

2. Support adding images to a gallery by only using the ~image_path~ parameter or
   the URL in the front matter such as:

   #+begin_src text
   gallery:
     - image_path: /assets/archive/image/foo/unsplash-gallery-image-1.jpg
     - /assets/archive/image/foo/unsplash-gallery-image-1.jpg
     - /assets/archive/image/foo/unsplash-gallery-image-1.jpg
   #+end_src

   A gallery originally requires both ~url~ and ~image_path~ in order to have a
   pop-up view of the images. This plugin just makes it more convenient to add
   images to a gallery. But you can still stick to the old method.

3. Support link abbreviations in image paths. See this [[#link-abbreviations][designated section]] for
   more details.

4. Use ~alt~ for ~title~ if ~title~ is unspecified so you don't need to write the same
   content for two parameters.

5. Support Liquid links in ~title~, e.g.,

   {% raw %}
   #+begin_src text
   ---
   gallery:
     - image_path: https://i.postimg.cc/Vv8jFw8D/unsplash-gallery-image-1.jpg
       title: "Prompt: [Local images]({{ site.baseurl }}{% post_url 2026-02-19-local-images %})"
   ---
   #+end_src
   {% endraw %}

All these customizations are backward-compatible, that is, you can still write a
gallery in the original way to get the original feel.

Here's a gallery demo utilizing the above features:

{% raw %}
#+begin_src text
---
link_abbrs:
  - link_abbr: foo https://i.postimg.cc/Vv8jFw8D/
gallery:
  - image_path: foo:unsplash-gallery-image-1.jpg
    alt: "Use alt for title and a link abbreviation."
  - image_path: foo:unsplash-gallery-image-1.jpg
    title: "Link to post: [Dark Mode]({{ site.baseurl }}/docs/dark-mode/)"
  - foo:https://i.postimg.cc/Vv8jFw8D/unsplash-gallery-image-1.jpg
  - foo:unsplash-gallery-image-1.jpg
---

{% include gallery columns=6 caption="My custom gallery" %}
#+end_src
{% endraw %}

* Link Abbreviations
#+LINK: foo https://i.postimg.cc/Vv8jFw8D/

You can use link abbreviations in a document drafted in Markdown or Org Mode,
for example, ~foo:my-image.jpg~ where ~foo:~ is a link abbreviation of the full path
of the parent of ~my-image.jpg~.

This lets you focus on resource names instead of their full paths when composing
links, and switch their parent path easily when many resources are under the
same parent.

To create a link abbreviation, in the front matter add the abbreviation and its
full form in a ~link_abbr~ in the ~link_abbrs~ array, like this:

#+begin_src org
link_abbrs:
  - link_abbr: foo https://i.postimg.cc/Vv8jFw8D/
#+end_src

Then you can use the abbreviation in links throughout the document. It will be
expanded to the full form in the built site.

Org Mode has its own built-in link abbreviations using ~#+LINK~. Here it can also
be used for Jekyll build. It acts as a fallback to ~link_abbrs~ in the front
matter, i.e., if an abbreviation isn't found in ~link_abbrs~ but found in ~#+LINK~,
also use it to expand links in the built site. ~#+LINK~ can appear anywhere in the
document except the front matter.

#+begin_src org
---
link_abbrs:
  - link_abbr: foo https://i.postimg.cc/Vv8jFw8D/
---

,#+LINK: foo https://your/fallback/link/
#+end_src

Below are the supported use cases of link abbreviations.

** Standard Org Links

Here is an image:

#+begin_src org
[[foo:unsplash-gallery-image-1.jpg]]
#+end_src

[[foo:unsplash-gallery-image-1.jpg]]

** Figure

Links written in Liquid syntax are also supported. For example, here is a
figure:

{% raw %}
#+begin_src org
{% include figure image_path="foo:unsplash-gallery-image-1.jpg" popup=true
alt="" caption="A figure" %}
#+end_src
{% endraw %}

{% include figure image_path="foo:unsplash-gallery-image-1.jpg" popup=true
alt="" caption="A figure" %}

** Gallery

You can create a gallery as usual in [[https://mmistakes.github.io/minimal-mistakes/docs/helpers/#gallery][Minimal Mistakes]]. The extra benefit is that
now you can use link abbreviations in it, like this:

* Loading Local Files

When running locally you can load resources on your computer by using symlinks.
Why symlinks? Because modern browsers block pages from =http(s)://‚Ä¶= from loading
local files via ~file://‚Ä¶~ for security (‚Äúcross-origin‚ÄĚ / local file access).

Follow the steps to use this feature:

1. Create a symlink to the target data in the ~symlinks:~ array in the front
   matter:

   #+begin_src org
   ---
   symlinks:
     - /assets/archive/image/foo ~/Downloads/temp/archive/image/foo
   ---
   #+end_src

   This creates the symlink =_site/assets/archive/image/foo= inside the site root,
   which points to the path =~/Downloads/temp/archive/image/foo/= on your
   computer.

2. Then you can refer to the files inside that path like this:

   {% raw %}
   #+begin_src markdown
   ---
   gallery:
     - url: /assets/archive/image/foo/20260218184820.png
       image_path: /assets/archive/image/foo/20260218184820.png
     - url: /assets/archive/image/foo/20260218184820.png
       image_path: /assets/archive/image/foo/20260218184820.png
   ---

   ![]({{site.url}}{{site.baseurl}}/assets/archive/image/foo/20260218184820.png)

   {% include gallery %}
   #+end_src
   {% endraw %}

You can even create a link abbreviation to the symlink. For example, here is a
gallery loading local files:

{% raw %}
#+begin_src org
---
symlinks:
  - /assets/archive/image/foo ~/Downloads/temp/archive/image/foo
link_abbrs:
  - link_abbr: foo2 /assets/archive/image/foo/
gallery3:
  - url: foo2:unsplash-gallery-image-1.jpg
    image_path: foo2:unsplash-gallery-image-1.jpg
    title: Here is a [link]({{ site.baseurl }}/docs/dark-mode/) in title.
  - image_path: foo2:unsplash-gallery-image-1.jpg
  - foo2:unsplash-gallery-image-1.jpg
  - image_path: [[foo2:unsplash-gallery-image-1.jpg]]
    title: An image referenced through an org link
  - [[foo2:unsplash-gallery-image-1.jpg]]
  - [[foo2:unsplash-gallery-image-1.jpg]]
---
